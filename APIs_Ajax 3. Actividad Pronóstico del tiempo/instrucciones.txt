Creemos una aplicación de pronóstico del tiempo en la que podamos ingresar cualquier ciudad y averiguar la temperatura en Fahrenheit. ¡Afortunadamente hay una API o un conjunto de URL que nos dirán la temperatura de cualquier ciudad! Utilizaremos la API Open Weather. Open Weather API nos brinda un conjunto de URL muy útiles que son divertidas de usar. Por ejemplo, navega al enlace   http://api.openweathermap.org/data/2.5/weather?q=London,uk&&appid= (asegúrate de agregar su API) ... y encontrarás datos JSON sobre Londres. De hecho, incluso si intentas navegar a http://api.openweathermap.org/data/2.5/weather?q=California,us&&appid= (con su ID de API) ..., la API le encontrará el mismo JSON datos. Esto significa que tenemos que validar menos las respuestas de diferentes usuarios en nuestra aplicación. Gracias, Open Weather API!

Vamos a tener un formulario donde nuestro usuario puede ingresar una ciudad en un formulario y mostrar la temperatura de la ciudad en la pantalla. Escucharemos el envío de un formulario para que podamos recopilar lo que el usuario ha completado y agregarlo a nuestra URL  http://api.openweathermap.org/data/2.5/weather?q= . Por ejemplo, si el usuario ha escrito 'Los Ángeles', agregaríamos esta entrada a  http://api.openweathermap.org/data/2.5/weather?q= para que la URL termine siendo  http: // api .openweathermap.org / data / 2.5 / weather? q = Los Ángeles. ¡Y no olvides agregar la clave API en la última parte de la URL!

Podemos escuchar un formulario enviado con la función submit () de jQuery . Sin embargo, la función submit () vuelve a cargar automáticamente la página, por lo que vamos a devolver false. Ahora que sabemos cómo hacer AJAX, no queremos que nuestra página se vuelva a cargar: queremos que los datos nos lleguen y luego podemos actualizar dinámicamente las partes que deben actualizarse en nuestra página actual. Este debería ser un buen punto de partida para configurar su jQuery para construir esta aplicación.
